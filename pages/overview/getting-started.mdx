## Use existing modules

Reuse existing modules with the [ModuleSDK](/module-sdk), a typeScript library for using smart account modules in applications and wallets.

Access an extensive set of [Core Modules](/module-sdk#core-modules) built by modules and a growing library of third-party modules.

Core Modules include base smart account features like Passkeys, Social Recovery, and MFA (multisig), as well as more opinionated features like Scheduled Order and Deadman Switch.

ModuleSDK is built to be coupled with:

permissionless.js | Biconomy SDK | ZeroDev SDK | userop.js

## Build a module

Build a smart account-agnostic module with [ModuleKit](/modulekit), a developer framework for building, testing, and deploying modules with Foundry.

## Create smart account automation

Build account automations using modules and the [Rhinestone Automations](/automations) service, a system that enables developers to effortlessly set up, manage, and automate smart account actions via modules.

## Get your module audited

[Contact us](http://t.me/kurt_larsen) to get matched with an auditor who can audit and attest to your module’s security on the Module Registry, a permissionless and ownerless system for scalable trust delegation when using modules.
